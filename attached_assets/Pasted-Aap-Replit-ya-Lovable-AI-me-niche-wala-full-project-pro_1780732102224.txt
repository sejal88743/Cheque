Create a mobile-first Progressive Web App (PWA) called "Cheque Entry Manager".

TECH STACK:

* React + TypeScript
* Vite
* Tailwind CSS
* Supabase (existing project connection)
* PWA enabled with offline support, installable on Android
* Responsive mobile UI
* Print-ready A4 reports

SUPABASE INTEGRATION:

* Connect with existing Supabase project.
* Read Bills data from existing tables.
* Do not create duplicate customer master data.
* Party Name, Cheque No and Bill information must automatically fetch from existing Supabase records using Bill No.

====================================
PAGE 1 : CHEQUE ENTRY
=====================

Fields:

1. Entry Date

   * Default current date.
   * Auto filled.
   * Editable.

2. Cheque Date

   * User enters only DD.
   * Month and Year auto use current month and year.
   * Example:
     User enters 15
     Auto convert to 15/06/2026.

3. Bill No

   * Searchable.
   * Multiple bill numbers allowed.
   * Example:
     7909+7908+7910

4. Party Name

   * Auto fetch from Supabase using Bill No.

5. Cheque Amount

6. Cheque No

   * Auto fetch and save against Bill No.
   * Editable if required.

7. Bank Name

   * Searchable dropdown.
   * Type 2-3 letters and suggestions appear.
   * Example:
     HDF -> HDFC BANK
     SBI -> STATE BANK OF INDIA

Bank Master table:

* HDFC BANK
* STATE BANK OF INDIA
* ICICI BANK
* AXIS BANK
* BANK OF BARODA
* UNION BANK
* CANARA BANK
* PUNJAB NATIONAL BANK
* IDBI BANK
* KOTAK MAHINDRA BANK

Features:

* Save
* Edit
* Delete
* Duplicate cheque detection
* Validation

Duplicate Rules:
Same:

* Cheque Date
* Cheque Amount
* Party Name
* Cheque No

Then show warning:
"Possible Duplicate Cheque Entry"
